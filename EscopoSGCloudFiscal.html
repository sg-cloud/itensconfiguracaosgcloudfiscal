    - Cabeçalho:

    > Organização logada;

    > Dt. Emissão (obrigatório): input de data, formato [dd/mm/aaaa]. Por default, preencher com a data atual da máquina;

    > Dt. Competência: adicionado essa informação no requisito pois no Análise tem, mas verificar se será necessário de fato;

    > UF Incidência (obrigatório): select com as siglas das UFs;

    > Cidade Incidência (obrigatório): select com as cidades da UF selecionada;

    > Cliente: abrir modal para selecionar o cliente. Adicionar também opção de cadastrar;

    > Botões: Salvar e Histórico. Botão salvar, permitir salvar os dados digitados, e salva a nota com situação Pendente, permitindo que a mesma seja editada posteriormente. Botão Histórico, retorna para a listagem da NFS-e. Caso o usuário tenha começado a digitação da nota, exibir mensagem de confirmação, se deseja salvar ou não os dados.

    > Input Serviço (obrigatório): busca os serviços cadastrados no módulo de Serviços. Listar apenas Serviços. Adicionar opção de cadastrar um serviço novo;

    > Cód. Serviço: input bloqueado. Trazer o que estiver preenchido no Cadastro do Serviço selecionado;

    > Quantidade (obrigatório);

    > Vlr. Serviço (obrigatório);

    > Vlr. Desconto;

    > Vlr. Total Serviço: valor do serviço - desconto;

    > Descrição do Serviço (obrigatório): varchar (2000). Limitar a digitação no input conforme a quantidade do banco (tag tsDiscriminacao);

    > Adicionar um ícone com tooltip informativo para a descrição do serviço. Informação do tooltip: "Informe uma descrição para o serviço prestado. Máximo de 2000 caracteres.";

    > Quantidade;

    > Vlr. Unitário;

    > Vlr. Total;
